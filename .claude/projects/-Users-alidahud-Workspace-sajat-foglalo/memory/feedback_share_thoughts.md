Always share thoughts before making any changes to the code.

**Why:** User wants to discuss and validate before edits happen, not just blindly execute.

**How to apply:** When the user asks for a change, share your take on it first (is it a good idea? any concerns? alternatives?), then make the change only after the user confirms.
